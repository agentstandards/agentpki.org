AgentPKI — 信任锚（JWKS / 撤销 / Status List）

配合“推理收据/验签”叙事，服务安全与合规人群。

JWKS 说明与轮换

  • 每把密钥设置 kid；轮换：先上新钥、后下旧钥（保留宽限期）。
  • 公开 jwks_uri，支持缓存（ETag/Cache‑Control）。
  • 参考：IETF RFC7517、Auth0/Okta 文档。

示例：jwks.json

撤销策略（CRL / Status List）

  • 并行支持：传统 CRL（简单清单）+ W3C Status List 2021（压缩位图）。
  • 常见撤销理由：所有权变更、密钥泄露、滥用、主动退出。
  • 响应建议：24h 内更新清单并公告时间戳。

模板与示例

// CRL (JSON)
{ "revoked": [ {"kid":"2025-01-01-rsa","reason":"compromise","revoked_at":"2025-01-10T00:00:00Z"} ] }
